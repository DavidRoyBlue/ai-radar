ICM (Interpreted Context Methodology) replaces framework-level orchestration with filesystem structure — folders become agent boundaries, files become memory and routing rules. It's the maximally minimal opposite of LangChain-style code-defined graphs, and it composes naturally with file-reading agents like Claude Code. Worth reading even if you don't adopt it because it forces you to articulate which parts of an agent system *need* code vs. which are happier as conventions on disk.
